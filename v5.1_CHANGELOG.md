# v5.1 CHANGELOG - 视觉模型 + 素材理解 + 智能剪辑

## 发布时间
2026-04-06

## 核心升级

### 1. 视觉理解模型 (MiniCPM-V / Qwen-VL)
- **功能**: 自动分析视频帧内容，理解产品、动作、场景
- **模型选择**: 
  - Qwen-VL-2B (推荐，开放模型)
  - MiniCPM-V-2_6 (需HuggingFace授权)
- **安装路径**: D:\AI Models\

### 2. 素材自动索引生成
- **功能**: 扫描素材目录，自动生成Excel索引
- **输出格式**:
  | 字段 | 说明 |
  |------|------|
  | 文件名 | 原始视频文件名 |
  | 产品名称 | 视频中产品名称 |
  | 镜头类型 | 特写/全景/动作等 |
  | 关键词标签 | 3-5个关键词 |
  | 时长(秒) | 视频时长 |
  | 是否可用 | 是/否 |

### 3. 素材直接使用
- **改变**: 不再强制使用AI生成片段
- **新流程**: 素材镜头 → 人工筛选/AI辅助筛选 → 智能拼接
- **优势**: 真实产品镜头，保留原始质感

### 4. MiniMax API 脚本生成
- **API配置**: 已配置在环境变量
- **温度参数**: 0.5 (稳定输出)
- **功能**: 基于参考文案生成爆款脚本

## v5.1 完整工作流

```
1️⃣ 视频素材扫描
    - 扫描 C:\Users\Administrator\Desktop\素材01
    - 使用 Qwen-VL 分析每帧内容
    - 生成 素材索引.xlsx

2️⃣ 人工校正索引
    - 打开 Excel 检查识别结果
    - 修正错误标签
    - 标记"是否可用"

3️⃣ 读取参考文案
    - 路径: 素材01\爆款文案参考.xlsx
    - 分析文案风格、结构

4️⃣ MiniMax 生成脚本
    - 调用 MiniMax API (temperature=0.5)
    - 生成10条口播脚本
    - 输出 hotspots/产品_scripts.json

5️⃣ 素材智能筛选
    - 根据脚本内容匹配素材
    - 选择最佳镜头组合
    - 避免重复使用同一镜头

6️⃣ 自动化剪辑
    - 方式A: 剪映/CapCut API
    - 方式B: FFmpeg 拼接 (降级方案)

7️⃣ 中英字幕 + 中文配音
    - FFmpeg 烧录字幕
    - TTS 生成配音

8️⃣ 质检输出
    - FFmpeg QC 检测
    - 输出 final_videos/
```

## 已知限制

1. **MiniCPM-V-2_6 需要授权**: 首次使用需要HuggingFace账号申请
2. **GPU显存要求**: Qwen-VL-2B 约需4GB显存，GT 710 (2GB) 可能不足
3. **CPU降级运行**: 显存不足时自动降级到CPU，速度较慢

---

## 降级方案

| 场景 | 降级方案 |
|------|----------|
| MiniCPM-V授权失败 | 使用 Qwen-VL-2B |
| Qwen-VL显存不足 | 使用 CPU 模式 (慢10x) |
| 剪映API失败 | FFmpeg 拼接 |
| TTS失败 | 仅字幕，无配音 |
