## Instructions

### 完整流程（9步）

```
1️⃣ 品牌分析 → 2️⃣ 热点搜索 → 3️⃣ 精准产品图查找 → 4️⃣ 脚本生成 → 5️⃣ 镜头生成 → 6️⃣ 自动化剪辑 → 7️⃣ 字幕配音同步 → 8️⃣ 成片评估 → 9️⃣ 输出报告
```

### Step 1: 解析用户输入

- 提取产品关键词（如"西梅汁"、"椰汁水"）
- 提取热点主题（如未提供，从热门趋势中选择）
- 如都未提供，调用 Brand-Analyzer 随机选择核心卖点

### Step 2: 调用 Brand-Analyzer

```
检索产品核心卖点（最多3个）
输出：产品ID、名称、卖点列表、推荐视频风格
```

### Step 3: 调用 Trend-Searcher

```
搜索当前最相关的热点趋势
输出：前5个热点话题，按热度排序
```

### Step 3.5: 调用 Product-Image-Searcher（新增）

```
确认产品外观：
1. 搜索该产品的官方高清图片（天猫/京东/官网/小红书）
2. 列出搜索链接，由用户确认并下载 3-5 张主图
3. 保存到 knowledge_base/products/{产品名}/images/
4. 将产品图路径加入脚本JSON，供 Clip-Generator 参考
输出：产品图片路径列表
```

### Step 4: 调用 Script-Writer

```
结合卖点和热点，生成：
- 15-30秒完整脚本
- 分镜头时间码
- 每个镜头的AI视频提示词（英文）
输出：保存到 metadata/scripts/{timestamp}_script.json
```

### Step 5: 调用 Clip-Generator

```
对每个镜头的提示词：
1. 查重（7天内相似提示词 → 复用）
2. 生成新镜头（最多重试2次）
3. 质量评估（<60分 → 丢弃）
输出：[合格镜头路径列表]
```

### Step 6: 调用 Video-Editor

```
输入：镜头文件列表 + 可选背景音乐
输出：
- 拼接后的成片
- 唯一追踪码（MD5）
- 更新镜头元数据
```

### Step 6.5: 调用 Subtitle-Audio-Sync（新增）

```
精准同步字幕与配音：
1. Whisper 对配音音频进行词级时间戳转写
2. 生成精确到毫秒的 SRT 字幕文件
3. 烧录硬字幕到成片（FFmpeg 或 MoviePy）
4. 输出同步报告
输出：带硬字幕的成片 + SRT字幕文件
```

### Step 7: 调用 Final-QC

```
输入：成片路径 + 脚本原文
输出：
- 多维度质量评分
- 综合得分
- 评估结论
- 报告保存路径
```

### Step 8: 输出报告

## 错误处理

| 步骤 | 失败原因 | 处理方式 |
|------|----------|----------|
| 产品图查找 | 搜索无结果 | 提示用户手动提供图片 |
| 镜头生成 | API超时/配额用尽 | 中止，通知用户 |
| 镜头质量 | 3次重试均不合格 | 中止，通知用户 |
| 视频剪辑 | FFmpeg错误 | 中止，通知用户 |
| 字幕同步 | Whisper失败 | 跳过同步，保留原字幕 |
| 成片评估 | API不可用 | 跳过评估，标记pending |

## 使用示例

**用户：** `用VideoProducer生产一个关于"西梅汁"的视频，热点就用"肠道健康"`

**执行过程：**
```
1️⃣ 品牌分析：轻上西梅汁 - 0糖、膳食纤维、东南亚原料
2️⃣ 热点搜索：#肠道健康 #养生 #好物推荐
3️⃣ 产品图查找：找到5个来源链接，用户确认下载3张
4️⃣ 脚本生成：22秒脚本已保存
5️⃣ 镜头生成：5个镜头已生成（2个复用，3个新生成）
6️⃣ 视频剪辑：拼接完成
6️⃣ 字幕同步：Whisper词级时间戳 → SRT已烧录
7️⃣ 质量评估：82/100 (A) - 良好
✅ 视频生产完成！
```

**最终输出：** (完整报告如上)

## 环境要求

- MiniMax API Key (环境变量: MINIMAX_API_KEY)
- ChromaDB (本地向量数据库)
- FFmpeg (视频处理)
- Python库: chromadb, sentence-transformers, moviepy, requests, faster-whisper
